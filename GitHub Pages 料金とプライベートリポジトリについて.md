# GitHub Pages 料金とプライベートリポジトリについて

## 現状

**GitHub Freeプランでは、プライベートリポジトリからGitHub Pagesを使うには有料プラン（GitHub Proなど）へのアップグレードが必要です。**

- **パブリックリポジトリ**: 無料でGitHub Pagesが使える ✅
- **プライベートリポジトリ**: 有料プラン（GitHub Pro: $4/月）が必要 ⚠️

## 解決策

### 方法1: パブリックリポジトリに変更（推奨・無料）

#### メリット
- ✅ **完全無料**
- ✅ **すぐに使用開始可能**
- ✅ **セキュリティリスクが低い**（理由は後述）

#### セキュリティ面の考慮

このアプリは**コードを公開しても安全**です：

1. **データは端末内のみ**
   - すべてのデータは`localStorage`に保存
   - サーバーにデータが送信されることはない
   - コードを見られても、データは見られない

2. **機密情報がコードに含まれていない**
   - APIキーなどの秘密情報は含まれていない
   - 設定（メール宛先など）は各端末で個別に設定

3. **静的サイト**
   - サーバーサイドの処理がない
   - データベースへの接続がない

#### リポジトリをパブリックに変更する方法

1. GitHubリポジトリの「Settings」タブを開く
2. 一番下の「Danger Zone」セクションまでスクロール
3. 「Change visibility」をクリック
4. 「Make public」を選択
5. リポジトリ名を入力して確認
6. 「I understand, change repository visibility」をクリック

### 方法2: 別の無料ホスティングサービスを使用（プライベートリポジトリのまま）

プライベートリポジトリのままで無料で使えるサービス：

#### Vercel（推奨）
- ✅ **プライベートリポジトリでも無料**
- ✅ **Next.jsの開発元が提供**（最適化されている）
- ✅ **自動デプロイ**（GitHubと連携）
- ✅ **カスタムドメイン対応**

#### Netlify
- ✅ **プライベートリポジトリでも無料**
- ✅ **簡単なデプロイ**
- ✅ **フォーム機能など追加機能あり**

### 方法3: 有料プランにアップグレード（非推奨）

- GitHub Pro: $4/月（年間$48）
- プライベートリポジトリでGitHub Pagesが使える
- ただし、このアプリの特性を考えると不要

## 推奨される選択

### 🏆 推奨: パブリックリポジトリ + GitHub Pages（無料）

**理由：**
1. **完全無料**
2. **セキュリティリスクが極めて低い**
   - データは端末内のみ
   - コードを見られてもデータは見られない
3. **すぐに使用開始可能**
4. **メンテナンスが簡単**

### 代替案: プライベートリポジトリ + Vercel（無料）

プライベートリポジトリのまま使いたい場合：
- Vercelはプライベートリポジトリでも無料
- GitHub Pagesと同様に簡単にデプロイ可能
- Next.jsに最適化されている

## よくある質問

### Q: コードを公開しても大丈夫ですか？

A: **はい、大丈夫です。**
- データは端末内のみで、サーバーに送信されません
- コードを見られても、データは見られません
- 機密情報（APIキーなど）は含まれていません

### Q: パブリックリポジトリに変更したら、後でプライベートに戻せますか？

A: **はい、可能です。**
- いつでもプライベートに戻すことができます
- ただし、プライベートに戻すとGitHub Pagesが使えなくなります（有料プランが必要）

### Q: プライベートリポジトリのまま無料で使いたい場合は？

A: **VercelまたはNetlifyを使用してください。**
- どちらもプライベートリポジトリで無料
- GitHub Pagesと同様に簡単にデプロイ可能

## 結論

**パブリックリポジトリに変更することを推奨します。**

理由：
- ✅ 完全無料
- ✅ セキュリティリスクが極めて低い
- ✅ すぐに使用開始可能

プライベートリポジトリのまま使いたい場合は、VercelまたはNetlifyを検討してください。
