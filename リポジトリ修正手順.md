# リポジトリの修正手順

## 問題の状況

- `tavi-ct-input`フォルダの中に、さらに`tavi-ct-input`という同名のフォルダがある
- 内側のフォルダに`.git`があり、そこがGitHubリポジトリと連携している
- 外側のフォルダ（実際のファイルがある場所）がGitリポジトリになっていない

## 解決方法（リポジトリ削除不要）

### ステップ1: 内側のフォルダの`.git`を削除

PowerShellで以下を実行：

```powershell
cd "C:\Users\sqjyp\OneDrive\デスクトップ\management\tavi-ct-input\tavi-ct-input"

# .gitフォルダを削除（リポジトリの接続を解除）
Remove-Item -Recurse -Force .git
```

### ステップ2: 外側のフォルダをGitリポジトリとして初期化

```powershell
cd "C:\Users\sqjyp\OneDrive\デスクトップ\management\tavi-ct-input"

# Gitリポジトリを初期化
git init

# すべてのファイルを追加
git add .

# 初回コミット
git commit -m "Initial commit: TAVI CT入力アプリ"
```

### ステップ3: 既存のGitHubリポジトリと接続

内側のフォルダで確認したリモートURLを使用：

```powershell
# [リモートURL]を実際のURLに置き換える
git remote add origin https://github.com/[ユーザー名]/[リポジトリ名].git

# メインブランチに設定
git branch -M main

# プッシュ（既存のリポジトリを上書き）
git push -u origin main --force
```

**注意**: `--force`オプションを使用するため、既存の`.gitattributes`ファイルは上書きされます。

### ステップ4: 内側の空のフォルダを削除（オプション）

```powershell
cd "C:\Users\sqjyp\OneDrive\デスクトップ\management\tavi-ct-input"

# 内側のtavi-ct-inputフォルダを削除
Remove-Item -Recurse -Force tavi-ct-input
```

## GitHub Desktopを使う場合

### 方法1: 既存のリポジトリを削除して再追加

1. GitHub Desktopで内側の`tavi-ct-input`フォルダのリポジトリを削除
2. 「File」→「Add Local Repository」
3. 外側の`tavi-ct-input`フォルダを選択
4. 「Create a new repository」を選択
5. 既存のGitHubリポジトリと接続

### 方法2: 手動で修正

1. 内側の`tavi-ct-input`フォルダの`.git`を削除（上記のステップ1）
2. GitHub Desktopで外側の`tavi-ct-input`フォルダを開く
3. 「Repository」→「Repository Settings」→「Remote」で既存のリモートURLを設定

## 確認方法

修正後、GitHubリポジトリページで以下を確認：

- `app/`フォルダが表示されている
- `components/`フォルダが表示されている
- `lib/`フォルダが表示されている
- `.github/workflows/deploy.yml`が表示されている
- `package.json`が表示されている

## まとめ

- ✅ リポジトリを削除する必要はありません
- ✅ 設定変更で対応可能です
- ✅ 内側の`.git`を削除して、外側のフォルダをGitリポジトリとして初期化
- ✅ 既存のGitHubリポジトリと接続してプッシュ
